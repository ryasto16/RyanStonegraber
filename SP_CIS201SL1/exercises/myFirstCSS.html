A First Look at CSS

Cascading Style Sheets (CSS) are designed primarily to separate document content (html code) from document presentation (styles). There are advantages to using CSS. At first you may find it confusing, but once you understand the rules of the cascade, you will appreciate the beauty of using CSS.

You should strive to use external or embedded style sheets as much as possible. Find the commonalities between formats and set the styles at the external or embedded level. Keep inline styles to a minimum. Inline styles should be the "exception", not the "rule".

Advantages of CSS

  • greater control
  • separates style from structure
  • reduce repetition of html code
  • easier site maintenance
  • improved accessibility

Rules of "Cascading"

Generally speaking, styles "cascade" into a new virtual style sheet by the following rules, where #4 has the highest priority:

  1. browser default
  2. external style sheet
  3. embedded styles (inside head)
  4. inline styles (inside individual tags)
